2、普通参数与基本注解

2.1、注解：

@PathVariable（ 接收请求路径中占位符的值 ）

处理请求 url 路径中的参数 /user/{id}

@RequestParam （用于将请求参数区数据映射到功能处理方法的参数上 ）

localhost:8080/**/?userName=zhangsan

value：参数名字，即入参的请求参数名字，如username表示请求的参数区中的名字为username的参数的值将传入；

required：是否必须，默认是true，表示请求中一定要有相应的参数，否则将报404错误码；

defaultValue：默认值，表示如果请求中没有同名参数时的默认值，

@RestController 等价于@Controller + @ResponseBody

@ResponseBody (返回json格式)

@RequestBody（请求参数以json格式提交）

基于注解配置Bean:

Spring 针对Bean管理中创建对象提供注解

@Component：可以用于注册所有bean

@Service：主要用于注册服务层的bean

@Controller：主要用于注册控制层的bean

@Repository：主要用于注册dao层的bean

上面四个注解功能是一样的，都可以用来创建bean实例

注入属性：

常用注解：

  1. @Autowired ：根据属性类型进行自动装配。 （常用）

  2. @Qualifier ：根据属性名称进行注入，与@Autowired连用 ，当注入的接口有多个实现类时,根据名字指定某个具体的实现类

  3. @Resource ：可以根据类型注入（默认），也可以根据名称注入。（它是java提供的注解，不是

Spring）

  1. @Value ：注入普通类型属性

mybats-plus配置文件

扫描MyBatis Mapper 所对应的 XML 文件位置，如果在 Mapper 中有自定义方法（XML 中有自定义实现），需要进行该配置，告诉 Mapper 所对应的 XML 文件位置，Maven 多模块项目的扫描路径需以 classpath: 开头 （即加载多个 jar 包下的 XML 文件）*

MyBaits 别名包扫描路径，通过该属性可以给包中的类注册别名，注册后在 Mapper 对应的 XML 文件中可以直接使用类名，而不用使用全限定的类名（即 XML 中调用的时候不用包含包名）。
